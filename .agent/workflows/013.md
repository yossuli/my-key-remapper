# [Workflow-013] AI 応答の日本語化徹底チェックリスト

## 概要

AI アシスタント（私）がユーザーに対して出力するメッセージ（チャット応答、Tool Call の引数、ログ、コミットメッセージなど）において、英語が混入することを防ぎ、**「完全な日本語対応」** という最上位ルールを遵守するための自己点検手順です。このワークフローが呼ばれたら以降そのセッションにおいてこのルールを守ってください。

## Bad / Good Example

### Bad Example (NG)

- **TaskSummary**: "I successfully Refactored the code." (英語)
- **TaskStatus**: "Updating task.md" (英語)
- **Log**: "Implemented feature X." (英語)
- **Chat**: "了解しました。I will proceed." (日英混在)

### Good Example (OK)

- **TaskSummary**: "コードのリファクタリングを完了しました。"
- **TaskStatus**: "task.md を更新中"
- **Log**: "機能 X を実装。"
- **Chat**: "了解しました。進めます。"

※ コード内の識別子（変数名、ファイル名）、ライブラリ名、専門用語（React, TypeScript など）は英語のままで構いません。

## Step-by-Step (自己点検手順)

ツール実行前、特に `task_boundary`, `notify_user`, `write_to_file` (Log/Review) を呼び出す **直前** に以下のチェックを行います。

1.  **Task Boundary チェック**:

    - `TaskName`: 日本語になっているか？（"Planning" -> "計画フェーズ" またはカタカナ "プランニング"）
      - ※ 慣習的に英語が適している場合も、可能な限り日本語を添える。
    - `TaskSummary`: **完全な日本語文** になっているか？ 思考ごとの翻訳漏れがないか？
    - `TaskStatus`: 次のアクションが **日本語** で記述されているか？

2.  **Notification チェック**:

    - `Message`: ユーザーへの問いかけや報告が自然な日本語か？

3.  **Logs / Artifacts チェック**:

    - `log.md` への書き込み内容は日本語か？
    - `implementation_plan.md` などのドキュメント作成時に、見出しや説明文が英語になっていないか？

4.  **引用・要約**:
    - ユーザーの英語メッセージを引用する場合を除き、要約や応答は日本語に翻訳する。
    - ステップの実行履歴を振り返る際も、英語のログをそのままコピペせず、日本語で要約し直す。

## チェックリスト

- [ ] `task_boundary` の引数（Summary, Status）が日本語であることを確認した。
- [ ] `notify_user` のメッセージが日本語であることを確認した。
- [ ] 生成・更新するドキュメント（ログ含む）が日本語であることを確認した。


## 知見の種 (Seed) の記録

運用を通じて得られた「コーディングパターンの改善案」や「ルールの例外事例」は、`seeds.md` (Working Dir) へ記録してください（[Workflow-seed] 準拠）。

## 知見の継続的改善 (Continuous Knowledge Improvement)

このワークフローを使用していても英語の混入が発生した場合は、なぜチェックをすり抜けたかを分析し、チェック項目や確認タイミングを修正してください。
