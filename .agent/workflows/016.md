# [Workflow-016] コンポーネント作成 (Create Atomic Component)

## 概要

React コンポーネントを新規作成する際の、Atomic Design に基づく分類基準、命名、実装ルールです。

## 分類基準 (Atomic Design)

- **Atoms**: ボタン、入力欄、アイコンなど、これ以上分割できない最小単位。状態を持たないことが多い。
- **Molecules**: Atoms の組み合わせ。検索フォーム（入力欄+ボタン）など。
- **Organisms**: 独立して機能する UI のセクション。ヘッダー、リストエディタなど。ドメイン知識を持つ。
- **Templates/Pages**: レイアウトやページ全体。

## 実装ルール

### 1. ディレクトリと命名

- パス: `src/renderer/src/components/[type]/[ComponentName].tsx`
- ファイル名: PascalCase (例: `MainButton.tsx`)
- Props 型名: `[ComponentName]Props` (export する)

### 2. Props 設計

- **Destructuring**: 引数内での分割代入を推奨します (`Workflow-004`)。
- **JSDoc**: 複雑な Props には説明を記述します。

### 3. Storybook / Test (Optional)

- 再利用可能な `Atoms`, `Molecules` については、Storybook ストーリーの作成を検討します。
- ロジックを含む場合は単体テストを作成します。
