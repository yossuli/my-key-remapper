# [Workflow-024] 意図の自律的補完 (Autonomous Complementation / Complete the Thought)

## 概要

ユーザーの断片的な指示（0.1->1）を、AI の自律的な具現化能力で完全な手順（1->10）に展開するためのプロセスです。ユーザーの I/O（タイピング・説明）コストを最小化しつつ、AI が「背景・ルール・手順」を完璧に補完した計画を構築します。

## 背景 (Rationale)

- **ユーザー**: 優れた判断（1）を持つが、それを具現化するための詳細説明やタイピングがボトルネックとなっている。
- **AI**: 0->1 の発想にはレビューが必要だが、与えられた 1 の意図を既存ルールやコードベースと照合し、具体的な手順「10」に展開するのは自律的に可能である。
- **目的**: ユーザーが「考えの断片」を投げれば、AI が「完成された実行計画」を返し、ユーザーはそれをレビュー（承認）するだけで済む体制を確立する。

## 手順 (Step-by-Step)

### 1. 受信 (Input)

- ユーザーからの短い指示、あるいは抽象的な要望を受け取ります。

### 2. 派生と自律的補完 (Autonomous Extrapolation)

- 受け取った指示から、以下の要素を AI が自律的に調査・推測し、「10 の完全な実行計画」を構築します。
  - **背景・真意**: なぜその指示が必要か？（例：是正後にコミットしたい、リネームして整理したい等）
  - **関連ファイルの特定**: 修正が必要なファイル、影響を受けるコンポーネント、参考にするべき既存コード。
  - **適用ルールの探索**: 遵守すべき `GEMINI.md`, `RULE.md`, 各 `Workflow` の特定。
  - **実行手順の多段階展開**: 安全な破壊的変更（cp -> git mv 等）や品質チェックを含む全ステップ。

### 3. 「10」の提示と合意形成 (Verify the Plan)

- 実行に移る前に、構築した「10」を以下の **固定フォーマット** で提示し、承認（Atomic Approval）を得ます。

> #### [補完された指示の要約]
>
> - **背景・意図**: （ユーザーの指示から補完された目的）
> - **遵守ルール**: [GEMINI.md], [Workflow-XXX] など、今回適用する規約。
> - **具体的アクション**:
>   1. （実行するステップ 1）
>   2. （実行するステップ 2 ...）
> - **影響・副作用**: （ファイル変更、コマンド実行による影響）

### 4. 実行 (Execute)

- ユーザーの「y」などの承認を得次第、提示した手順に従って自律的に実行を開始します。
