# [Workflow-024] 意図の補完と具体化 (Complete the Thought)

## 概要

ユーザーの優れた発想（0->1）を、AI の高速な具現化能力で完全な形（1->10）にするためのプロセスです。ユーザーの I/O（タイピング・説明）コストを最小化し、AI が文脈と知識でその間を埋めます。

## 背景 (Rationale)

- **ユーザー**: 優れたアイデア（0->1）を持つが、それを表現する **I/O 速度（タイピング等）がボトルネック** となっている。
- **AI**: 0->1 の発想には試行錯誤が必要だが、与えられた 1 のアイデアを具体的な実装・手順（1->10）に展開するのは驚異的に高速である。
- **目的**: ユーザーは「考えの断片」を与え、AI がそこから派生して「10 の完全な形」を確認するステップを踏むことで、I/O の制約を超えて生産性を最大化する。

## 手順 (Step-by-Step)

### 1. 受信 (Input)

- このワークフローを使用するようにという指示とともに送られてきたメッセージについて考えます

### 2. 派生と補完 (Extrapolate)

- 受け取った指示から、以下の要素を AI が推測・補完し、「完全な手順（The "10"）」を構築します。
  - **背景意図**: なぜそれが必要か？（例：エディタの Diff 機能を使いたいから）
  - **具体的コマンド**: 何を実行すべきか？（例：`cp` ではなく `git add`）
  - **関連ルール**: 既存の `Workflow` や `GEMINI.md` との整合性。
  - **エッジケース**: ファイルがない場合は？ 新規作成時は？

### 3. 確認 (Verify)

- 実行に移る前に、構築した「10 の指示」をユーザーに提示し、合意形成を図ります。
  - **メッセージ例**:
    > 「ご指示の意図は〜いうことですね。具体的には以下の手順（1〜N）で実行するという理解で合っていますか？」

### 4. 実行 (Execute)

- ユーザーの「OK」「Go」などの承認を得てから、手順を実行します。
