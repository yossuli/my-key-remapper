description: プロジェクトで定義されている「よくある修正（Common Fixes）」の一覧を表示する
---

1. `.agent/workflows/` ディレクトリ内のファイルを `list_dir` で取得確認する。
2. 数字で始まるファイル（`00*.md`）を中心に、各ファイルの `description` (フロントマター) またはタイトルを読み取る。
3. 読み取った情報を元に、以下の形式でユーザーに提示する。

## よくある修正リスト (Common Fixes)

以下のコマンドを使用して、定型的な修正自動化ワークフローを実行できます。

| コマンド  | ファイル | 説明                             |
| :-------- | :------- | :------------------------------- |
| `@[/000]` | `000.md` | (ファイルを読み取って説明を記述) |
| `@[/001]` | `001.md` | (ファイルを読み取って説明を記述) |
| ...       | ...      | ...                              |

4. ユーザーに対して「実行したいワークフローがあれば、`@[/001]` のように指定してください」と促す。

## 知見の種 (Seed) の記録

運用を通じて得られた「コーディングパターンの改善案」や「ルールの例外事例」は、`seeds.md` (Working Dir) へ記録してください（[Workflow-seed] 準拠）。
